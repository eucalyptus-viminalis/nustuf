AGENT: OK here's the tldr:

**What is MPP?**

Machine Payments Protocol — an open standard for HTTP 402 ("Payment Required"). It lets any client (agent, app, human) pay for any service inline with the HTTP request itself. No API keys, no billing accounts, no checkout flows.

**How it works:**
1. Client requests a resource
2. Server returns `402` with a payment challenge
3. Client pays (on-chain, Stripe, whatever)
4. Client retries with payment proof in the header
5. Server verifies, delivers the resource

**Why you should care:**

This is basically what x402 does in nustuf, but as a formal IETF spec with multi-payment-method support. It's being pushed by Tempo (the company behind it) with SDKs in TypeScript, Python, and Rust.

**Opportunities:**
- nustuf already uses x402 — MPP is the standardized version of the same pattern. Could be worth aligning with it
- They have a **service directory** at mpp.dev/services — listing nustuf there = free discovery
- If nustuf supports MPP-compatible headers, any MPP client can buy from you automatically (not just nustuf agents)
- Currently only supports Tempo and Stripe as payment methods — someone could build an EVM/Base payment method adapter

Basically it's the protocol layer for what you're already building. Want me to dig deeper into how it compares to x402 specifically?

AGENT: Saved the research. Here's the comparison:

**x402 (Coinbase) — what nustuf uses now:**
- Coinbase-built, open source
- EVM-native (Base, Solana, any ERC-20)
- Uses `PAYMENT-REQUIRED` / `PAYMENT-SIGNATURE` custom headers
- Payment verified via a "facilitator" service (Coinbase-hosted)
- Has Permit2 for gasless payments
- SDKs: TypeScript, Go, Python

**MPP (Tempo) — the newer standard:**
- IETF spec attempt (`paymentauth.org`)
- Uses standard `WWW-Authenticate: Payment` / `Authorization: Payment` headers (proper HTTP auth framework)
- Payment-method agnostic (currently Tempo chain + Stripe)
- Tempo chain = ~500ms finality, sub-cent fees, purpose-built for micropayments
- Has "session" intent for streaming/metered payments (x402 doesn't)
- Service directory at mpp.dev/services

**Key differences:**

| | x402 | MPP |
|---|---|---|
| Chains | Base, Solana, any EVM | Tempo chain (own L1) |
| Currency | Any ERC-20 (USDC default) | Tempo stablecoins |
| Headers | Custom (`PAYMENT-REQUIRED`) | Standard HTTP auth (`WWW-Authenticate`) |
| Facilitator | Coinbase-hosted | None needed |

| Streaming payments | No | Yes (sessions) |
